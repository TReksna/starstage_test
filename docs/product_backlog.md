# Product Backlog (not built yet)

The research suggests several larger product directions. These are **intentionally not
implemented** in the test version — they are recorded here for later prioritization. Do not
build these without an explicit decision; the current focus is answer quality + feedback.

| Idea | Notes / rationale |
|------|-------------------|
| Full onboarding redesign | Smoother birth-data capture, consent, and privacy explanation up front. |
| Monetization system | Users distrust aggressive monetization — design carefully and transparently if pursued. |
| Subscriptions | Tiering (e.g. free plain answers, paid technical/depth). Must avoid hidden charges. |
| Live astrologer marketplace | Human astrologer booking/handoff. Large scope; trust + payments. |
| Remedy store | High distrust of remedy upsells; only if clearly optional and non-manipulative. |
| Notifications | Weekly themes / dasha-change nudges. Needs opt-in and restraint. |
| Journaling system | Emotionally-safe self-reflection log tied to readings. |
| Compatibility module | Two-chart synastry. Needs its own safety framing (no relationship doom). |
| Full dashboard redesign | Chart dashboards, saved readings, history views. |

## Near-term, low-risk follow-ups (candidates, still not scheduled)

- Privacy & methodology page (what data is stored, how charts are computed).
- Weekly theme summaries derived from current dasha/transits.
- Feedback review UI (read `answer_feedback` rows in the admin/Logs area).
- Per-user default style/depth preference persistence.
